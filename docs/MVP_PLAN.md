# MVP 开发计划

## Phase 1：架构与骨架（当前）

**目标：** 搭建可运行的项目框架，数据库可连接，前后端可通信

**任务清单：**
- [x] 项目架构设计
- [x] 数据库表设计
- [x] API 接口设计
- [x] 页面与组件拆分设计
- [x] 初始化前后端目录结构
- [ ] 编写 Docker Compose 配置
- [ ] 编写后端依赖与 Dockerfile
- [ ] 编写后端核心模块（main, config, database）
- [ ] 编写后端数据模型
- [ ] 编写后端认证路由骨架
- [ ] 编写后端课程路由骨架
- [ ] 编写前端 package.json 与配置文件
- [ ] 编写前端全局布局与导航
- [ ] 编写前端首页 Dashboard 骨架
- [ ] 编写环境变量模板

**验收标准：**
- `docker-compose up` 能启动数据库
- 后端 `/health` 接口返回 200
- 前端 `npm run dev` 能正常启动
- 前后端能正常通信

---

## Phase 2：用户系统 + 课程模块 + 学习进度

**目标：** 用户能注册登录，能看到课程列表，能浏览课时内容，系统记录学习进度

**任务清单：**
- [ ] 用户注册/登录/登出完整实现
- [ ] JWT 认证中间件
- [ ] 课程与课时种子数据（先 5 节课）
- [ ] 课程列表 API 与页面
- [ ] 课时详情 API 与页面
- [ ] 课时内容渲染（Markdown + 代码高亮）
- [ ] 学习进度记录 API
- [ ] 课时完成标记功能
- [ ] 首页 Dashboard 显示真实进度

**验收标准：**
- 新用户可注册并自动登录
- 用户能看到课程章节列表
- 用户能阅读课时教程
- 用户标记完成后进度更新

---

## Phase 3：练习题系统 + 代码运行沙箱

**目标：** 用户每节课学完能做配套练习题，编程题可在线运行

**任务清单：**
- [ ] 练习题种子数据（先 25 道题，覆盖前 5 节课）
- [ ] 6 种题型前端展示组件
- [ ] 练习题提交 API（选择/判断/填空）
- [ ] 代码运行沙箱 Docker 实现
- [ ] 编程题判题逻辑（运行用户代码 + 对比输出）
- [ ] 代码编辑器（Monaco）集成
- [ ] 运行结果展示
- [ ] 错题本记录
- [ ] 练习结果页面（正确/错误 + 解析）

**验收标准：**
- 选择/判断/填空题能判对错
- 编程题能在沙箱中运行
- 运行超时/内存超限能正确捕获
- 错题能自动记录到错题本

---

## Phase 4：AI 助手集成

**目标：** AI 能根据上下文答疑、解释报错、点评代码

**任务清单：**
- [ ] AI Provider 抽象层实现
- [ ] OpenAI Provider 实现
- [ ] 系统提示词设计（教学风格）
- [ ] AI 聊天 API 与前端界面
- [ ] AI 解释报错功能
- [ ] AI 代码点评功能
- [ ] AI 生成额外练习功能
- [ ] 课程上下文注入（AI 知道用户在学哪课）

**验收标准：**
- AI 回答符合教学风格（先类比再代码）
- AI 知道用户当前在学哪节课
- AI 能解释 Python 报错信息
- AI 不给完整答案，只给提示

---

## Phase 5：项目实战模块

**目标：** 用户能按步骤完成实战项目

**任务清单：**
- [ ] 项目与任务种子数据（1 个项目完整实现）
- [ ] 项目列表页面
- [ ] 项目详情与任务拆解页面
- [ ] 分步任务引导 UI
- [ ] 项目任务提交与判题
- [ ] 参考答案与 Hint 展开机制
- [ ] AI 项目代码点评

**验收标准：**
- 项目按任务拆解展示
- 用户完成一步后才能看下一步（或自由选择）
- 不一次性暴露全部答案
- AI 能点评项目代码

---

## Phase 6：首页 Dashboard + 数据统计

**目标：** 首页展示完整学习数据，提升用户粘性

**任务清单：**
- [ ] Dashboard 数据统计 API
- [ ] 学习天数计算
- [ ] 正确率统计
- [ ] 今日任务推荐算法（最简单的：推荐下一节未学课程）
- [ ] 学习进度可视化（环形图、进度条）
- [ ] 最近活动记录
- [ ] 错题本页面

**验收标准：**
- 首页能看到总进度百分比
- 能看到今日推荐任务
- 错题本能查看和重新练习
